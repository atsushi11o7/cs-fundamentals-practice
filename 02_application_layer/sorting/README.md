# ソート（Sorting）

並べ替えは頻出だが、実装で悩むのはたいてい「複数の基準で並べたいとき」。
そこを `key` とタプルで表現するのが定石になる。

## `sorted` と `key` の基本

`sorted(iterable, key=関数)` の `key` は「各要素を、比較に使う値に変換する関数」。
要素そのものではなく、`key` が返した値で大小比較される。
```python
sorted(users, key=lambda u: u.age)   # age で昇順
```

## 複合キー：タプルを返すのが定石

「主キーで並べ、同じなら副キーで並べる」はタプルで表現する。
タプルは先頭から順に比較されるため、`(第1基準, 第2基準)` を返せばよい。
```python
sorted(items, key=lambda x: (第1基準, 第2基準))
```

## 降順の表現

- 数値の降順：`-値` を key にする。**複合キーの一部だけを降順にできる**のが利点。
  ```python
  key=lambda x: (-x.score, x.name)   # score降順、name昇順
  ```
- 全体を降順：`sorted(..., reverse=True)`。ただし一部だけ降順にはできない。
- 文字列の降順は符号反転が使えないため、`reverse` や別の工夫が要る。

## 安定ソートという性質

Pythonのソートは安定（stable）。同じ key を持つ要素同士は元の順序が保たれる。
これを使えば「まず副キーでソート → 次に主キーでソート」と多段で並べることもできるが、
複合キーで一度に書く方が読みやすいことが多い。

## 設計上の判断とトレードオフ

- **key の中で重い計算をしない**：要素数ぶん実行されて遅くなる。
  事前に計算した値を持たせる（Decorate-Sort-Undecorate）などを検討する。
- **集計とソートは段階を分ける**：「集計してから並べる」ときは、
  まず辞書で集計し、次に `sorted(d.items(), key=...)` で並べる、と分けて考えると混乱しない。

## 計算量

比較ソートなので O(N log N)。`key` の計算が O(1) ならこれが支配的。
