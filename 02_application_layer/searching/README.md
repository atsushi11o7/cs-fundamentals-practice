# 探索（Searching）

線形探索・二分探索・全探索の考え方と使い分け。
ソート済みデータで二分探索がなぜ効くのか、そして「境界を求める探索」の書き方まで。

## 線形探索：素直だが O(n)

先頭から順に見て、目的の値を探す。ソートされていなくても使えるのが強み。
その代わり、最悪では全要素を見るので O(n)。データが小さい・一度きり・未整列なら十分。

Python では `x in seq` や `list.index(x)` が実質これ。**リストの `in` は O(n)** である点は
意識しておく（メンバー判定を大量に行うなら集合や辞書に持ち替える。→ `data_structures`）。

## 二分探索：ソート済みなら O(log n)

**ソート済み**であることを前提に、探索範囲を毎回半分に狭める。
中央要素と比べ、目的の値がどちら側にあるかで範囲を捨てていく。
n 個を毎回半分にするので、比較回数は log₂ n 回。100万件でも約20回。

前提が「ソート済み」なので、探索前にソート（O(n log n)）が要るなら、
一度きりの探索では線形の方が速い。**何度も探索するデータ**でこそ二分探索が生きる。

## 実装で必ず悩む「範囲の持ち方」

二分探索のバグはほぼ境界（off-by-one）で起きる。区間の持ち方を最初に固定すると崩れにくい。

- 本実装は**半開区間 `[lo, hi)`** で持つ（`hi` は「範囲の外側」を指す）。
- 空になる条件は `lo == hi`。ループ条件は `while lo < hi`。
- 「候補から外した」要素は範囲から外す（`lo = mid + 1`）／
  「まだ候補」の要素は残す（`hi = mid`）。この対応を混ぜないこと。

## 値そのものではなく「境界」を探す

実務で多いのは「ちょうどの値」より「◯◯以上が最初に現れる位置」を求める探索。
`lower_bound` はこれで、target が無くても**挿入すべき位置**を返す。

- ソート済み配列への挿入位置
- 「以上・より大きい・以下」の件数を数える
- しきい値を満たす最小の答えを二分探索で求める（答えで二分探索）

判定を `seq[mid] < target` に変えるだけで、等値探索から境界探索に変わる。
この一般化が二分探索の応用力の源になる。

## 標準の道具を使うか、自分で書くか

標準ライブラリの `bisect`（`bisect_left` = lower_bound、`bisect_right`、`insort`）が
まさにこれを提供する。実務では `bisect` を使ってよい。
ただし境界の詰め方を自分で書けることは、応用（答えで二分探索など）で効いてくる。

## 設計上の判断とトレードオフ

- **二分探索はソートが前提**。未整列なら線形か、事前ソートの元が取れるかで判断する。
- **一度きりなら線形、繰り返すなら二分**。ソートコストを回収できるかで選ぶ。
- **境界条件を仕様に合わせる**：`<` と `<=`（lower_bound と upper_bound）の取り違えが
  件数のズレに直結する。

## 計算量

- `linear_search`：O(n)
- `binary_search` / `lower_bound` / `contains`：O(log n)（ソート済み前提）
