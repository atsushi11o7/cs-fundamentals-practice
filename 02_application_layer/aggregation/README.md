# 集計（Aggregation）

「キーごとに値をまとめる」処理。データを扱うアプリケーションで最も頻繁に登場する操作で、
合計・カウント・グルーピングの3つが基本形になる。

## なぜ辞書（ハッシュマップ）を使うのか

集計は「キーを見て、対応する値を更新する」の繰り返し。
辞書はキーによる参照・更新が平均 O(1) で行える。
もしリストで「そのキーが既にあるか」を毎回探すと O(N) の走査が必要になり、
全体で O(N²) に悪化する。**集計といえば辞書**、という対応づけを基本に持つ。

## 「存在しないキー」をどう扱うか

集計を書くとき最初に判断が要るのがここ。
まだ登録されていないキーに `d[key] += value` すると KeyError になる。
対処は3通り、状況で使い分ける。

- `d.get(key, 0)`：キーが無ければ 0 を返す。合計・カウント向き。
  ```python
  totals[key] = totals.get(key, 0) + value
  ```
- `defaultdict(型)`：アクセス時に自動で初期値を作る。リストにためる（グルーピング）向き。
  ```python
  groups = defaultdict(list)
  groups[key].append(value)   # キーが無くても空リストから始まる
  ```
- `Counter`：カウント専用。最も簡潔。

## 標準の道具を使うか、自分で書くか

`Counter(items)` は1行でカウントできる。実務では積極的に使ってよい。
ただし「中で何が起きているか」を理解しておくことは別途重要。
`get(key, 0)` で自力集計を書ける理解の上で、簡潔な道具を選ぶのが望ましい。
便利な道具を使えることと、原理を理解していることは両立させる。

## 設計上の判断とトレードオフ

- **`defaultdict` を返すときは注意**：呼び出し側が普通の辞書を期待していると、
  存在しないキーへのアクセスで勝手にキーが増える挙動に驚くことがある。
  外部に返す前に `dict(...)` に変換すると安全。
- **キーにできるのはイミュータブルなもの**：文字列・数値・タプルは可、リストは不可（ハッシュできない）。

## 計算量

いずれも要素数 N に対して O(N)。辞書アクセスが O(1) だから全体も O(N) に収まる。
