# ケーススタディ

身近なサービスを題材に、01〜04 の各技術を組み合わせて設計・実装する。
各ケースは `design_method` の型（要件確認 → 機能洗い出し → データ設計 → 構成 →
スケール対応 → 運用監視）に沿って README を書き、ドメインの核を実装 + tests で確かめる。

1ケース = 1ディレクトリ。実装はドメインの核に集中し、キャッシュ・分散・レート制限などは
該当レイヤー（01〜04）の実装を参照する形にする。

## ケース一覧

- **url_shortener** — URL短縮サービス。連番＋base62 の採番、解決、クリック計測。
  読み取りが多いサービスの最適化（キャッシュ・シャーディング）を型に沿って設計。

（SNSタイムライン・EC在庫・チャットなどは今後追加予定。）
